## Задачи
### Последний четный элемент
Дан массив, состоящий из положительных чисел. Найти индекс последнего четного элемента. Если таких нет, то вернуть -1.
Реализуйте и протестируйте алгоритм.

### Генетическая
Цепочка ДНК состоит из четырёх нуклеотидов:
- аденин (A);
- цитозин (C);
- гуанин (G);
- тимин (Т).

Цепь РНК состоит также из четырёх нуклеодитов:
- A;
- C;
- G;
- Урацил (U).

Цепь РНК строится на основе цепи ДНК последовательной заменой комплементарных
нуклеотидов:

|ДНК|РНК|
|---|---|
| G  | C |
| C | G |
| T | A |
| A | U |

Реализуйте и протестируйте алгоритм, переводящий цепь ДНК в цепь РНК.
```cs
string DnaToRna(string rna)
```

### Сложение
Дано число, хранящееся в виде массива цифр `byte[] digits`. 
Реализовать и протестировать алгоритм добавления к этому числу 1.
```cs
byte[] inc(byte[] digits)
```
### Внешняя зависимость
Реализуйте и протестируйте метод, который считывает с консоли имя пользователя и
возвращает его, обрезав пробелы в начале и конце.
```cs
string ReadUserName()
```

## Ручное тестирование
Ручное тестирование предполагает выполнение тестов без использования специализированных инструментов автоматизации. 
### Упражнение
<details>
<summary>Техническое задание</summary>
**Функциональные требования**

**1. Поля формы**

**1.1 Поле "Email"**
- Тип: текстовое поле
- Обязательность: обязательное поле (помечено *)
- Валидация:
  - Поле не должно быть пустым
  - Пробелы в начале и конце должны автоматически удаляться (trim)
  - Поле, содержащее только пробелы, считается пустым
  - Email должен соответствовать формату: `username@domain.com`
- Сообщение об ошибке при пустом поле: "Email обязателен для заполнения"
- Сообщение об ошибке при неверном формате: "Введите корректный email"

**1.2 Поле "Пароль"**
- Тип: поле пароля (символы скрыты)
- Обязательность: обязательное поле
- Валидация:
  - Поле не должно быть пустым
  - Пробелы в начале и конце должны автоматически удаляться (trim)
  - Поле, содержащее только пробелы, считается пустым
  - Минимальная длина: 6 символов
  - Максимальная длина: не ограничена
- Сообщение об ошибке при пустом поле: "Пароль обязателен для заполнения"
- Сообщение об ошибке при недостаточной длине: "Пароль должен содержать минимум 6 символов"

---

2. Кнопка отправки
- Текст кнопки: "Зарегистрироваться"
- Действие: отправка формы и валидация данных

---

3. Поведение формы

**3.1 При успешной валидации:**
- Показать сообщение: "Регистрация прошла успешно! Добро пожаловать."
- Цвет сообщения: зеленый
- Очистить все поля формы

**3.2 При ошибках валидации:**
- Показать сообщения об ошибках под соответствующими полями
- Цвет сообщений об ошибках: красный
- Не очищать поля формы (оставить введенные данные)

---

**Нефункциональные требования**

**4. Интерфейс**
- Все тексты должны быть на русском языке
- Орфография и грамматика должны быть корректными

**Критерии приемки**
1. Все обязательные поля должны проверяться на заполненность
2. Email должен валидироваться по указанному формату
3. Пароль должен содержать минимум 6 символов
4. Сообщения об ошибках должны быть понятными и корректными
5. При успешной регистрации форма должна очищаться
6. Все тексты должны быть без орфографических и грамматических ошибок
7. Интерфейс должен быть интуитивно понятным
